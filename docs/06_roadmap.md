# 个人开发路线图

## 第 0 阶段：准备期，1 周

目标：确定定位和边界。

任务：

- 明确只做猫狗。
- 梳理 50 个高频问题。
- 找 1-2 位执业兽医或宠物医疗从业者做兼职审核。
- 申请域名。
- 确定技术栈。
- 设计数据表和页面草图。

交付：

- 产品原型。
- 内容库大纲。
- AI 回答模板。

## 第 1 阶段：MVP，3-4 周

目标：能让真实用户建档、提问、记录、设置提醒。

功能：

- 登录。
- 宠物档案。
- AI 问答。
- 症状分诊。
- 餐食建议。
- 健康记录。
- 提醒。
- 管理后台。

质量要求：

- 红色急症必须能稳定识别。
- AI 日志可追踪。
- 成本可统计。
- 用户能反馈“有帮助/没帮助/危险”。

## 第 2 阶段：内测，2-4 周

目标：50-200 个真实宠物主使用。

任务：

- 找朋友、社群、小红书用户内测。
- 每天看 AI 日志。
- 收集错误回答。
- 补知识库。
- 调整问答模板。
- 优化建档流程。

指标：

- 建档率。
- AI 问答率。
- 7 日留存。
- 用户反馈。
- 红色急症漏判数量。

## 第 3 阶段：公开测试，1-2 个月

目标：跑通内容获客和会员转化。

功能：

- 微信支付会员。
- AI 深度报告。
- 文章 SEO/H5。
- 小程序分享卡片。
- 就医摘要导出。
- 化验单解释。

运营：

- 每周 5-10 篇短内容。
- 每周 2-3 篇长尾搜索文章。
- 做 1 个主题社群。

## 第 4 阶段：商业化增强，3-6 个月

目标：让平台有稳定小收入。

功能：

- 兽医审核问答。
- 本地医院导流。
- 商品 CPS。
- 宠物保险推荐。
- 多宠家庭会员。
- 健康月报。

注意：

- 兽医服务要找有资质人员。
- 医疗建议和广告推荐必须明显区分。
- 商品推荐要优先基于宠物档案，不要粗暴卖货。

## 第 5 阶段：长期扩展

可选方向：

- 多模态识别：便便、皮肤、眼鼻分泌物、化验单。
- 智能硬件数据接入：饮水、喂食、体重、猫砂盆。
- 宠物医院 SaaS：问诊前信息采集、复诊提醒。
- 宠物保险风控数据。
- 宠物慢病管理。

## 优先级总表

| 优先级 | 功能 | 原因 |
| --- | --- | --- |
| P0 | 宠物档案 | 一切个性化的基础 |
| P0 | AI 问答 | 核心体验 |
| P0 | 风险分诊 | 健康安全底线 |
| P0 | 后台日志 | 质量控制 |
| P1 | 健康记录 | 留存和长期价值 |
| P1 | 提醒 | 高频复访 |
| P1 | 餐食建议 | 高需求、低履约成本 |
| P2 | 会员 | 商业验证 |
| P2 | 内容 SEO | 低成本获客 |
| P3 | 在线兽医 | 需要资质和履约 |
| P3 | 商城 | 容易分散精力 |
