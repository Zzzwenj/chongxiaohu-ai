# 宠小护 1.0 产品范围、UI/UX 与模型能力策略

日期：2026-04-27

## 1. 为什么现在显得单一

之前的 1.0 更像“AI 问答页”，这会带来三个问题：

- 用户只有生病时才打开，低频。
- 页面压力都在问诊上，医疗风险变高。
- 没有日常记录，AI 无法变得更懂这只宠物。

更合理的定位是“日常养宠工作台”，问诊只是其中一个模块。

## 2. 竞品启发

公开应用资料里，高频能力集中在：

- 喂食管理：餐次、热量、零食、食物标签。
- 遛狗记录：时间、路线、便便尿尿、活动趋势。
- 健康档案：疫苗、驱虫、体重、就诊、用药。
- 提醒系统：喂饭、遛狗、洗澡、美容、复诊、驱虫疫苗。
- AI 助手：症状初筛、喂养、行为、护理。
- 图片/视频：便便、皮肤、眼睛、食物标签、行为状态。

## 3. 首页该展示什么

首页第一屏应该展示：

- 当前宠物。
- 今日完成度。
- 下一项待办。
- 喂饭/遛狗/互动/健康记录提醒。
- AI 快捷入口。
- 红黄绿风险状态。

不应该第一屏展示：

- 大段医疗免责声明。
- 复杂疾病百科。
- 商城商品流。
- 社区动态。
- 医生排行榜。

## 4. 功能优先级

### P0：每天会用

- 今日喂食提醒。
- 遛狗/互动提醒。
- 饮水和排便排尿记录。
- 体重记录。
- 宠物档案。
- AI 快捷问答。

### P1：建立信任

- 症状分诊。
- 就医摘要。
- 驱虫疫苗提醒。
- 知识库来源展示。
- 用户反馈纠错。

### P2：形成壁垒

- 图片/视频分析。
- 食物标签分析。
- 健康趋势图。
- 多宠家庭。
- 家人协作。

### 暂不做

- 泛社区信息流。
- 自营商城。
- 在线处方。
- 未审核疾病百科。
- 复杂医生服务市场。

## 5. 图片/视频分析路线

当前 DeepSeek API 主要适合文本推理，不应假装已经能可靠读图。图片/视频能力建议分三步：

1. 上传图片/视频 + 用户描述，先做文本分诊。
2. 接入视觉模型，识别便便、皮肤、眼睛、食物标签、行为异常。
3. 把视觉结果转成结构化标签，再交给宠物知识库和大模型综合回答。

推荐视觉模型候选：

- 通义千问 VL。
- 豆包视觉理解。
- 智谱 GLM-4V。
- 第三方 OCR/图像识别服务。

## 6. “训练专家级宠物大模型”的正确路径

不建议一开始训练模型。先建立这套闭环：

```text
知识库
  -> RAG 检索
  -> 大模型回答
  -> 用户反馈
  -> 人工/兽医审核
  -> 更新知识库
  -> 加入评测集
  -> 必要时微调模型
```

专家级能力来自：

- 高质量宠物知识库。
- 红黄绿分诊规则。
- 真实问题和错误案例。
- 兽医审核内容。
- 评测集。
- 多模态结构化标签。

后期可以微调：

- 回答格式。
- 追问方式。
- 中国养宠语境。
- 就医摘要生成。
- 危险问题拒答。

不应该用未审核用户反馈直接训练模型。

## 7. 1.0 页面调整

上一轮已把用户端从单问答页改为：

- 顶部品牌与快捷动作。
- 今日照护英雄区。
- 宠物档案。
- 今日提醒。
- AI 专家助手。
- 图片/视频分析入口。
- 回答与反馈区。
- 展示/暂不展示的产品取舍提示。

## 8. 本轮 UI/UX 细化

本轮进一步把产品改成“养宠工作台”，而不是“问诊工具”。

### 小程序当前结构

- 顶部宠物状态：品牌、物种、年龄、当前宠物名。
- 今日照护主视觉：完成度、下一项、体重、风险、待办数量。
- 四个主入口：今日、记录、AI、影像。
- 今日：时间线式提醒，适合喂饭、遛狗、互动、健康记录。
- 记录：进食、饮水、便便、体重和宠物档案。
- AI：症状、餐食、作息和就医摘要。
- 影像：图片/视频入口，为后续视觉模型接入预留。
- AI 回答区：只有用户触发问答后才展示，不抢占首页。

### 为什么这样设计

- 用户每天打开时，第一件事不是问病，而是照顾宠物。
- “问诊”改成“AI 专家助手”，降低医疗化压力。
- 影像入口提前占位，为后续便便、皮肤、眼睛、食物标签分析做准备。
- 记录模块是长期壁垒，因为每只宠物的历史上下文会越来越完整。

## 9. 页面展示规则

### 默认展示

- 今日待办。
- 当前宠物状态。
- 下一项提醒。
- 快速记录。
- AI 快捷问题。
- 媒体分析入口。
- 风险等级。
- 知识库来源。

### 条件展示

- AI 回答：用户提问后显示。
- 红色急症提示：命中风险规则后置顶。
- 遛狗任务：只对狗展示。
- 媒体预览：用户上传后展示。

### 暂不展示

- 泛社区动态。
- 商城商品瀑布流。
- 医生排行榜。
- 复杂疾病百科。
- 未审核用户经验。
- 处方药推荐。

## 10. 专家模型路线

“专家宠物级别”不能只靠换一个模型，需要四层能力：

1. 宠物知识库：疾病、喂养、行为、护理、急症、禁忌。
2. 结构化档案：物种、年龄、体重、绝育、慢病、饮食、历史记录。
3. 安全规则：红色急症、人药、毒物、处方药、延误就医风险。
4. 评测与反馈：错误回答、用户反馈、兽医审核、回归测试。

后续训练顺序：

- 先做 RAG 和规则。
- 再做高质量评测集。
- 再做回答风格和追问方式微调。
- 最后才考虑针对宠物场景的模型微调。

不要直接用用户反馈训练模型，必须先审核、脱敏、归类。
