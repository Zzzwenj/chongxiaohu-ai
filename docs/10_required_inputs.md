# 1.0 上线前需要你提供的资料

这些内容我已经在系统里做了占位，不影响本地运行；但要真实上线，需要你逐步补齐。

## 必须提供

- 产品名称：已定为“宠小护”。
- Logo 和品牌色：已有第一版 SVG 占位，可继续微调。
- 小程序主体：个人、个体工商户或公司。
- 微信小程序 AppID：用于替换 `apps/miniapp/src/manifest.json` 里的占位值。
- 域名。
- ICP 备案信息。
- 大模型供应商：已按 DeepSeek 接入。
- 大模型 API Key、Base URL、模型名：本地 `.env` 已可配置，不要提交到代码仓库。
- 隐私政策里的主体名称、联系邮箱。
- 用户协议主体名称。

## 强烈建议提供

- 1-2 位执业兽医或宠物医疗从业者，用于审核高风险知识。
- 第一批可信资料来源链接。
- 不允许 AI 回答的药物清单。
- 你希望优先服务的物种：第一版建议只做猫狗。
- 是否开启云同步：默认本地优先，可选云同步。

## 后续商业化需要

- 微信支付商户号。
- 会员价格。
- 客服微信或邮箱。
- 合作医院/医生资料。
- 商品 CPS 平台或淘宝客/京东联盟/精选联盟账号。
- 发票和经营主体信息。

## 当前占位说明

- AI 回答：未配置真实模型时使用 mock 逻辑。
- 知识库：内置少量示例知识，来源为占位，不能直接作为正式医疗内容上线。
- 用户数据：用户端宠物档案保存在浏览器 localStorage。
- 数据库：当前后端使用内存数组，重启会清空对话和反馈；生产版需要接 PostgreSQL。
